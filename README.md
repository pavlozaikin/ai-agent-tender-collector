# Tender Agent — збір тендерів з автохімії (PROZORRO)

AI-агент, який щодня переглядає українську систему держзакупівель **PROZORRO**,
знаходить **нові** тендери, пов'язані з автохімією, і надсилає звіт електронною
поштою з прямими посиланнями для перевірки.

**Категорії автохімії:** охолоджувальні рідини та антифризи, гальмівні рідини,
рідини для омивача скла, моторні / індустріальні / базові оливи.

Один і той самий тендер у звіт двічі не потрапляє — агент запам'ятовує вже
надіслані тендери.

---

## Як це працює

Конвеєр (LangGraph), один запуск = один цикл звіту:

```
crawl ─→ prefilter ─→ classify ─→ dedupe ─→ render ─→ notify ─→ persist
PROZORRO  широкий     LLM —        прибрати  HTML-     e-mail    зберегти
feed      фільтр      фінальне     вже       звіт      SMTP      курсор +
          (CPV +      рішення      надіслані                     надіслані
          ключові     щодо
          слова)      релевантності
```

- **crawl** — інкрементальний обхід стрічки `/tenders` від збереженого курсора.
- **prefilter** — широкий, навмисно надлишковий фільтр (групи CPV + ключові
  слова з `config/filters.yaml`). Лише здешевлює потік перед LLM.
- **classify** — LLM приймає **остаточне рішення** про релевантність.
- **dedupe** — відкидає тендери, надіслані в попередніх запусках (SQLite).
- **render / notify** — формує HTML-звіт українською та надсилає його поштою.
- **persist** — зберігає надіслані тендери та курсор стрічки.

Кожен виклик LLM логуються в таблицю `llm_usage` (токени + орієнтовна вартість).

## Вимоги

- **Docker** + **Docker Compose** (рекомендований спосіб запуску), або
- **Python 3.12** + [`uv`](https://docs.astral.sh/uv/) для локального запуску.
- Ключ API хоча б одного LLM-провайдера (OpenAI / Anthropic / Google / Perplexity).
- Доступ до SMTP-сервера (Gmail або корпоративна пошта).

## Налаштування

### 1. Змінні середовища — `.env`

```bash
cp .env.example .env
```

Відредагуйте `.env`. Ключові групи (повний опис — у `.env.example`):

- **LLM-моделі** — два призначення, кожне з основною та резервною моделлю:
  - `classify` — масовий, дешевий (рекомендовано `openai:gpt-5.4-mini`);
  - `report` — малий обсяг, якісніший (рекомендовано `openai:gpt-5.5`).
- **Ключі провайдерів** — `OPENAI_API_KEY` тощо. Потрібні лише ті, що
  використовуються.
- **SMTP** — `SMTP_HOST/PORT/SECURITY/USERNAME/PASSWORD/FROM`. Працює як з
  Gmail (App password), так і з будь-яким корпоративним сервером.
- **Розклад** — `SCHEDULE_CRON` (за замовч. щодня о 08:00 за Києвом).

> Моделі задаються у форматі `провайдер:модель`. Перед використанням звірте
> назви моделей із чинним каталогом провайдера.

### 2. Отримувачі звіту — `recipients.yaml`

```bash
cp config/recipients.example.yaml recipients.yaml
```

Відредагуйте `recipients.yaml` — списки `to` / `cc` / `bcc`. Файл має
коментований шаблон. Агент перечитує його перед кожним запуском.

> **Важливо:** створіть `recipients.yaml` **до** `docker compose up`, інакше
> Docker створить замість файлу теку.

### 3. Фільтр (за бажанням) — `config/filters.yaml`

Широкі групи CPV та ключові слова. Редагуйте, щоб розширити чи звузити
охоплення — без зміни коду.

`.env`, `recipients.yaml` та тека `data/` — у `.gitignore` і ніколи не
потрапляють до репозиторію.

## Запуск через Docker (рекомендовано)

```bash
docker compose up -d        # запускає планувальник (щоденний cron)
docker compose logs -f      # перегляд логів
docker compose exec tender-agent tender-agent run --now   # разовий запуск зараз
docker compose down         # зупинка
```

База даних SQLite зберігається у volume `tender-data`.

## Запуск локально (uv)

```bash
uv sync                                  # встановити залежності
uv run tender-agent run --now            # один цикл зі звітом
uv run tender-agent run --now --dry-run  # без надсилання e-mail і без запису стану
uv run tender-agent schedule             # запустити планувальник (блокуючий)
uv run tender-agent stats --usage        # статистика використання LLM
uv run tender-agent healthcheck          # перевірка стану
```

`--dry-run` виконує весь конвеєр, зберігає HTML-звіт у `data/reports/`, але
**не надсилає пошту і не змінює стан** — безпечно для тестування.

> Перший запуск переглядає тендери за останні `CRAWL_LOOKBACK_DAYS` днів —
> це може зайняти кілька хвилин. Наступні запуски обробляють лише новину.

## Розробка

```bash
uv sync
uv run ruff check .          # лінтер
uv run ruff format .         # форматування
uv run mypy src              # перевірка типів (strict)
uv run pytest                # тести (unit + integration) + покриття

pre-commit install           # хуки якості перед кожним комітом
```

CI (GitHub Actions, `.github/workflows/ci.yml`) виконує лінт, перевірку типів і
тести на кожен push / PR.

### Структура

```
src/tender_agent/
  settings.py          конфігурація (.env)
  state.py             моделі домену + стан LangGraph
  storage.py           SQLite: курсор, дедуплікація, llm_usage
  filters.py           широкий префільтр (CPV + ключові слова)
  llm.py               LLM-шар: провайдери, fallback, облік токенів
  pipeline.py          конвеєр LangGraph (вузли + граф)
  main.py              CLI + планувальник
  prozorro/            клієнт і моделі API PROZORRO
  emailer/             SMTP-надсилач, завантажувач отримувачів, HTML-звіт
```

## Безпека

- Секрети — лише в `.env` (ніколи не комітиться).
- `recipients.yaml`, `data/`, бази даних і робочі документи — у `.gitignore`.
- Docker-образ працює від непривілейованого користувача.
- Перед першим комітом перевірте: `git status` не показує `.env`,
  `recipients.yaml` чи `data/`.
